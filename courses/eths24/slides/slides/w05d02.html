HTML/CSS

HTML/CSS style uses hierarchies, specifically a specificity hierarchy.

  • CSS styling can be more or less specific
  • In case of contradiction, more specific wins

We can use this to:

  • Apply style broadly
  • Make local design choices
  • Impose special conditions based on page conditions

Consider the following HTML snippet:

<b>Some coffee <span style="font-weight:normal">isn't</span> bold but some is.</b>

Some coffee isn't bold but some is.

HTML and CSS

HTML elements and CSS rules are similarly vertically structured.

HTML is a

  • Element
    • Start Tag
      • Attributes
        • Attribute value
        • Attribute name
    • Content
    • End Tag
  • Element
    • Void Tag
      • Attributes
        • ...

CSS is a

  • Rule
    • Selector(s)
    • Declaration Block
      • Declaration
        • Property
        • Value
  • Attribute Value
    • Declaration Block
      • ...

Functionalities

Introduce the media attribute.

  • In a style element, we can set the media attribute to print.
    • <style type="text/css" media="print">
  • We can then use specificity to change how a page will print.
    • a {
	color: black;
	text-decoration-line: none;
}
  • We could change font-types, from monitor-friendly to print-friendly, colors to print in grayscale, etc.
  • We can use accessibility features, such as for braille devices.
  • We can use custom layouts, such as for handhelds and televisions.

Style elements

Consider the following CSS excerpt with a style element. 

<!DOCTYPE html>
<html lang="en">
	<head>
		<title>Sample page</title>
		<link rel="stylesheet" href="./styles.css"/>		
		<style type="text/css" media="print">
			.page_outer{
				box-shadow: none;
			}
			a {
				color: black;
				text-decoration-line: none;
			}
		</style>
	</head>
  • Recall: Specificity wins.
    • We have a general CSS stylesheet for web-render
    • We have a specific style element for print media
