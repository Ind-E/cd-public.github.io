textToNum

I implemented textToNum recursively.

  • For a string of length one, we convert using utf8ToInt
  • For a string of greater than length one:
    • textToNum the rest of the string is
    • find the value of the current character.
    • We multiply by 0x100 - the number of possible suffix/prefixes - and add the suffix/prefix
textToNum <- function(s)
{
  val = substring(s,1,1)
  val = utf8ToInt(val)
  cs = nchar(s)
  if (cs > 1)
  {
    rst = substring(s,2,cs)
    rst = textToNum(rst)
    val = val + 0x100 * rst
  }
  return(val)
}

I implemented numToText recursively.

  • For zero, return nothing.
  • For non-zero:
    • Find the value modulo 0x100

numToText <- function(n)
{
  !n && return()
  v = n %% 0x100
  r = n /  0x100
  r = floor(r)
  v = intToUtf8(v)
  r = numToText(r)
  return(paste0(v,r))
}

Problem space

So we need to differentiate C and I. We will define them mathematically.

  • We will use the notion of a public-key cryptosystem.

    A public-key cryptosystem is a technique to convert message of up to fixed size (that is, with a maximum value) to "cyphertext" - a numerical value that contains the information of a message, but not in a readable manner. To be a public-key system, any agent must be able to produce cyphertexts, but only the intended receiver may extract the original message from the cyper text.

    A public-key is a numerical code used to encrypt a message such that it can only be decrypted using a private key.

By far the most well-known and widely used public-key cryptosystem is RSA (Rivest–Shamir–Adleman),

Key Insight

There exists a key insight that underpines all of RSA. It based on the idea of modular arithmetic.

  • For some message m
  • Some encryptor e (the public key)
  • Some decryptor d (the private key)

We can find m,e, d and novel value n such that: 

((m ^ e) ^ d) %% n == m
  • m is the message.
  • m ^ e is the cypher text
  • n is the maximum value for m

Usefulness

Given some keyGen

  • I find some encryptor e (the public key)
  • I find a corresponding decryptor d (the private key)
  • I post e publicly.
  • Anyone who wishes to send me a private message may do so
    • They write a message.
    • They textToNum the message.
    • They raise the numerical value to the power e to get the cypher.
    • They send me the cypher without concern that others may read it.

Usefulness

Given some keyGen

  • I receive a cypher made using e (the public key)
  • I know the decryptor d - a value no one else knows.
    • I raise the cyper to the power d
    • I divide by n and take the remainder.
    • I numToText the remainder.
    • I read the message.
