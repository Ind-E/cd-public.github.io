Entropy

Or "entropy bits" if a physicist is around.

The entropy of a password is the base-2 logarithm of the number of guesses needed to find the password with certainty. A password with an entropy of 42 bits calculated in this way would be as strong as a string of 42 bits chosen randomly, for example by a fair coin toss.

One way to think of this, is if you started listing all possible passwords in order (say, beginning from "a"), how long would it take until you get to a given password?

Flipping Coins

Take a fair coin C s.t. P(C → H)=0.5

  • We flip the coin n times.
  • The resultant sequence (not set) has n bits of entropy.

Sequences

Take a fair coin C s.t. P(C → H)=0.5

  • We flip the coin 3 times.
  • The 8 possible sequences capture 3 bits of entropy.
  • 2^3=8
seqs <- list("HHH","HHT",
             "HTH","HTT",
    	     "THH","THT",
    	     "TTH","TTT")

Password Generation

Restrict passwords the 26 lower case letters of the 26-letter modern Latin alphabet

  • We flip the coin 5 times.
  • The 32 possible sequences capture 5 bits of entropy.
  • 2^5=32
  • 32 > 26 & 26 > (32 / 2)

  • L ⊂ C5 → {a,b,c,d,e,f,g,h,i,j,k,l,m,n,o,p,q,r,s,t,u,v,w,x,y,z}
  • We use a dictionary.

          "HHHTT"="d",...

Bits

In computers, we don't have heads or tails.

  • We have bits, which are variables that take a value in {0,1}.
  • L ⊂ {0,1}5 → {a,b,c,d,e,f,g,h,i,j,k,l,m,n,o,p,q,r,s,t,u,v,w,x,y,z}
  • We can take "H" to denote 0.
labs <- c("00000"="a",
          "00001"="b",
          "00010"="c",
          "00011"="d",... 
          "11010"="z",
          "11011"=NULL,...)

Bit Encoding

A bit string {0,1}n is just an integer.

  • We can show the equivalence with a dictionary
  • L ⊂ {0,1}n → ℤ: integers
labs <- c("00000"=0,
          "00001"=1,
          "00010"=2,
          "00011"=3,...

Bit Encoding

A bit string {0,1}n is just an integer.

  • We can show the equivalence with a function
  • L ⊂ {0,1}n → ℤ: integers
numToBits <- function(val) {
  paste(tail(rev(as.integer(intToBits(val))), n=5), collapse="")
}

Now in Binary

Representing Integers

  • The number of symbols available to count with determines the base of a number system
    • Decimal is base 10, as we have 10 symbols (0-9) to count with
      • Each new number counts for 10 times as much as the previous
    • Binary is base 2, as we only have 2 symbols (0 and 1) to count with
      • Each new number counts for twice as much as the previous
  • Can always determine what number a representation corresponds to by adding up the individual contributions

Specifying Bases

  • So the binary number 00101010 is equivalent to the decimal number 42
  • We distinguish by subsetting the bases: \[ 00101010_2 = 42_{10}\]
  • The number itself still is the same! All that changes is how we represent that number.
    • Numbers do not have bases – representations of numbers have bases

BONUS: Other Bases

  • Binary is not a particularly compact representation to write out.
    • Octal (base 8) uses the digits 0 to 7
    • Hexadecimal (base 16) uses the digits 0 to 9 and then the letters A through F
    • Decimal (base 10) uses the digits 0 to 9 but really isn't any fun at all


  • Why octal or hexadecimal over our trusty old decimal system?
    • Both are powers of 2, so it makes it easy to convert back to decimal

Representation

  • Sequences of bits have no intrinsic meaning!
    • Just like a series of coin flips.
  • You can use bits to represent any value you could imagine!
    • Characters represented by numeric character codes
    • Two-dimensional arrays of bits representing images
    • Sequences of images representing video, etc
  • To be useful though, everyone needs to agree on a representation!

Lower Case

First "pqlaksiwuejdhfyryt" and "correcthorsebatterystaple"

  • How many bits does each character require?
  • Does it matter if the letters are composed into words?

We can fit a lower case letter into ~5 bits - log2(26) ~= 4.7

lowCaseEntropy <- function(str) {
  nchar(str) * log2(26)
}

Why can we use a non-integer number of bits here?

c(lowCaseEntropy(ps[3]),lowCaseEntropy(ps[4]))

Letters and Numbers

Next "piis31415926"

  • How many bits does each character require?
  • 26 letters and 10 digits gives 36 unique characters...
    • this is a valid base 36 numeral!

We can fit both into ~5 bits - log2(5) ~= 5.17

goodEntropy <- function(str) {
  nchar(str) * log2(85)
}
