Entropy

Or "entropy bits" if a physicist is around.

The entropy of a password is the base-2 logarithm of the number of guesses needed to find the password with certainty. A password with an entropy of 42 bits calculated in this way would be as strong as a string of 42 bits chosen randomly, for example by a fair coin toss.

One way to think of this, is if you started listing all possible passwords in order (say, beginning from "a"), how long would it take until you get to a given password?

Flipping Coins

Take a fair coin s.t. P(H)=0.5

  • We flip the coin n times.
  • The resultant sequence (not set) has n bits of entropy.

Sequences

Take a fair coin s.t. P(H)=0.5

  • We flip the coin 3 times.
  • The 8 possible sequences capture 3 bits of entropy.
  • 2^3=8
