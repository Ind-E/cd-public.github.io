Problem space

So we need to differentiate C and I. We will define them mathematically.

  • We will use the notion of hashing.

    Hashing is a technique to convert numerical value of arbitrary size (that is, with no maximum value) to a numerical value of a fixed size (that is, with some maximum value).

  • The most basic form of hashing is modular arithmetic, or taking remainders. 
    min_hash <-function(n, max)
{
  return(n %% max)
}

Hashing

Multiple values may hash to the same value. 

min_hash(11,10) == min_hash(21,10)

One-way hashing is associated with integrity, similar to signatures.

A hash-based signature is a technique were a numerical value of small size is computed from a large amount of numerical data in such a manner that, if the overall data is changed, the hash is highly likely to change.

  • Hash-based signatures are often used to allow us to validate executables online.
  • Programs are numerical data, and we wish to downloaded unaltered programs for security purposes.
  • Signatures, specifically "SHA256SUMS" can be seen when downloading Ubuntu.

Use case

Consider you wish to have a new operating system.

  1. You navigate to ubuntu.com and download a file.
    • We assume you trust Ubuntu, but do you trust your connection? The file itself?
  2. You additionally download a hash-based signature.
  3. On your own device, which you trust, you compute what the hash-based signature of your downloaded file is.
    • We can imagine Ubuntu ran hash("ubuntu.exe"), and we do the same.
    • If the hash is the same, there is only a 1-in-hashmax chance the code was altered.
  4. If the hashes are the same, open the file.

This is how we verify the integrity of a file!

Representation

  • Sequences of bits have no intrinsic meaning!
    • Just like a series of coin flips.
  • You can use bits to represent any value you could imagine!
    • Characters represented by numeric character codes
    • Two-dimensional arrays of bits representing images
    • Sequences of images representing video, etc
  • To be useful though, everyone needs to agree on a representation!

Representing Characters

  • Use numeric encodings to represent character data inside the machine, where each character is assigned an integer value.
  • Character codes are not very useful unless standardized though!
    • Competing encodings in the early years made it difficult to share data across machines
  • First widely adopted character encoding was ASCII (American Standard Code for Information Interchange)
  • Originally just with 128 possible characters, even after expanding to 256, ASCII proved inadequate in the international world, and has therefore been superseded by Unicode.

ASCII

Meeting intToUtf8 and utf8ToInt

  • Python includes two build in functions to simplify conversion between an integer and the corresponding Unicode character
  • intToUtf8 takes an integer and returns the corresponding Unicode-8 character as a string
    • intToUtf8(65) gives "A" (capital A)
  • utf8ToInt goes the other direction, taking a single character string and returning the corresponding integer of that character in Unicode
    • utf8ToInt("B") gives 66
    • utf8ToInt(" ") gives 32

Abstract Strings

  • Characters (and their Unicode representation) are most often used in programming when combined to make collections of consecutive characters called strings.
  • Internally, strings are stored as a sequence of characters in a sequential chunk of memory.
  • You don’t have to (and generally don’t want to) think of the internal representation.
    • Better to think of the string as a single abstract unit
  • R emphasizes this abstract view by defining a built-in string class that already defines a selection of higher-level operations on string objects

We will call this function num_to_text. 

num_to_text <- function(n)
{
  retStr = ''
  char1 = as.integer(n / (2 ^ 8))
  retStr = paste(retStr, intToUtf8(char1), sep='')
  char2 = n - (char1 * (2 ^ 8))
  retStr = paste(retStr, intToUtf8(char2), sep='')
  return(retStr)
}

Modulus

Instead, we use modulo arithmetic, which in R is expressed using %% 

num_to_text <- function(n)
{
  retStr = ''
  char1 = as.integer(n /  (2 ^ 8))
  retStr = paste(retStr, intToUtf8(char1), sep='')
  char2 = as.integer(n %% (2 ^ 8))
  retStr = paste(retStr, intToUtf8(char2), sep='')
  return(retStr)
}
